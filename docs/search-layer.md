# Search Layer

## Overview

Search layer для каталога Black Green — простой, быстрый, MVP-friendly.

## Принятые решения

### Что ищем
- Название товара
- Подзаголовок (subtitle)
- Категория
- Ключевые слова из описания
- Характеристики (для чая: вкус, аромат; для посуды: материал, назначение; для подарков: состав, получатель)

### Что НЕ ищем
- Полный текст описания (тяжело, медленно)
- Теги или сложные атрибуты
- Ценовой диапазон (отложено)

## Архитектура

```
src/lib/search.ts     — логика поиска
src/components/
  SearchInput.tsx     — поле ввода
  SearchResults.tsx   — результаты + пустая выдача
  CatalogSearch.tsx   — обёртка с состоянием
```

### Voice Input (Future)

Search layer уже подготовлен для голосового ввода:

1. **SearchInput** принимает `onVoiceClick` callback
2. **checkVoiceSupport()** проверяет Web Speech API
3. **normalizeVoiceInput()** нормализует результат распознавания

Для добавления voice:
- Подключить Web Speech API или внешний сервис (Yandex SpeechKit, Google Cloud Speech)
- Передавать результат в тот же `onChange`, что и текстовый ввод
- Переиспользовать ту же логику поиска

## Пустая выдача (Empty State)

Аккуратная, дружелюбная:
- Иконка поиска
- Сообщение «Ничего не найдено»
- Кнопка очистки
- Кнопка перехода в каталог
- Подсказки по типам поиска

## Производительность

- Поиск по in-memory массиву (без API)
- Мгновенная фильтрация на клиенте
- debounce не требуется (данные локальные)

## Ограничения MVP

- Нет fuzzy search (орфография должна быть точной)
- Нет поиска по нескольким категориям сразу
- Нет сортировки результатов
- Нет "похожих товаров" в результатах
